Tips

Key Configuration Files

inputs.conf

Defines what Splunk reads or listens to.

Applied on UFs for file, TCP, and UDP inputs; on the indexer and HF for Splunk-to-Splunk receiving ports.
outputs.conf

Defines where a forwarder sends events next.

Applied on the direct UF to send to the indexer, on the via-HF UF to send to the heavy forwarder, and on the HF to send parsed data to Splunk Enterprise or Splunk Cloud target groups.
Splunk Cloud forwarding

A single HF can route different inputs to different outputs.

Separate HFs are optional, but often used when Cloud traffic needs separate certificates, network paths, scaling, change control, or failure isolation.
props.conf

Defines event parsing behavior such as timestamps, line breaking, and sourcetype handling.

Applied where parsing occurs: on the indexer for direct UF data, and on the heavy forwarder before it forwards parsed events.
transforms.conf

Defines regex-based event rewrites, including masking sensitive values.

In this lab, the HF applies transforms to `lab:masked` before forwarding to the indexer, so stored events contain masked email and card values.
